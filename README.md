# MaxContainer

Изолированная среда для запуска мессенджера **MAX** с жёсткими ограничениями доступа к системе.

## Возможности контейнера

- изоляция процессов  
- запрет доступа к устройствам  
- отсутствие аудио/видео  
- ограниченная файловая система (`~/.config/MAX`)  
- отсутствие дополнительных прав
- отсутствие DBus-привилегий  
- минимальные возможности взаимодействия с системой  

---

## Как собрать образ и запустить MAX

1. Скачайте [Dockerfile](https://github.com/AKotov-dev/MaxContainer/blob/main/Dockerfile) из этого репозитория  
2. Установите [DockerManager (rpm/deb)](https://github.com/AKotov-dev/docker-manager/releases) и загрузите в него `Dockerfile`:  
   **ПКМ → Создать образ из Dockerfile → Файлы проекта → Добавить файлы**
3. Введите имя создаваемого образа **max**  
4. Нажмите **ОК** и дождитесь завершения сборки (вес сборки ~700 MB) 
5. Скачайте ярлык [MAXContainer.desktop](https://github.com/AKotov-dev/MaxContainer/blob/main/MAXContainer.desktop) и запустите его

Настройки мессенджера сохраняются в каталоге: `~/.config/MAX`

---

## Удаление

Для удаления образа и контейнеров воспользуйтесь скриптом [remove-max.sh](https://github.com/AKotov-dev/MaxContainer/blob/main/remove-max.sh).

Каталог `~/.config/MAX` удаляется только с правами root (если требуется полная очистка).

---

## Мотивация

MAX — закрытое проприетарное приложение с широким доступом к окружению пользователя.  
Этот контейнер создаёт безопасную, ограниченную среду, уменьшая возможное воздействие приложения на систему и ваши данные.

MAX **не включён** в образ.  
Он загружается автоматически в момент сборки из официального источника.
